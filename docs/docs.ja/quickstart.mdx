**Apache Superset を試す準備はできましたか?** このクイックスタート ガイドは、**3 つの簡単な手順** でローカル マシン上で起動して実行するのに役立ちます。
[Docker](https://www.docker.com)、[Docker Compose](https://docs.docker.com/compose/)、[Git](https://git-scm.com/) がインストールされていることを前提としています。

:::caution
サンドボックス型環境での迅速な開始やその他の開発型ユースケースには `Docker Compose` の使用をお勧めしますが、**本番環境でのこの設定はお勧めしません**。
この目的については、[Kubernetes へのインストール](/docs/installation/kubernetes/) ページを参照してください。
:::

### 1. Superset を入手

```bash
git clone https://github.com/apache/superset
```

### 2. Superset の最新公式リリースで開始する

```bash
# Enter the repository you just cloned
$ cd superset

# Set the repo to the state associated with the latest official version
$ git checkout tags/4.1.1

# Fire up Superset using Docker Compose
$ docker compose -f docker-compose-image-tag.yml up
```

Docker Compose が基盤となるコンテナ イメージを取得し、いくつかの例を読み込むため、少し時間がかかる場合があります。
すべてのコンテナがダウンロードされ、出力が安定したら、ログインする準備が整います。

⚠️ 「validating superset\docker-compose-image-tag.yml: services.superset-worker-beat.env_file.0 must be a string」のようなエラー メッセージが表示される場合は、`docker-compose` のバージョンを更新する必要があります。
`docker-compose` は廃止予定であるため、代わりに `docker compose` を使用する必要があります。

### 3. Superset にログイン

次に、[http://localhost:8088](http://localhost:8088) にアクセスし、デフォルトで作成されたアカウントでログインします:

```bash
username: admin
password: admin
```

#### 🎉 おめでとうございます! これで、Superset がマシン上で起動し、実行されました。 🎉

### まとめ

Superset の使用が完了したら、他のコンテナ環境と同様に停止して削除できます:

```bash
docker compose down
```

:::tip
Superset はデータをローカルに保持するため、同じ環境を複数回使用できます。
ただし、Docker Compose の `stop` コマンドを実行して、すべてのプロセスを適切に停止してください。
そうすることで、データの破損や損失を回避できます。
:::
