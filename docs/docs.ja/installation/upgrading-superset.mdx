# Superset のアップグレード

## Docker Compose

まず、Docker Compose で実行中のコンテナをシャットダウンしてください:

```bash
docker compose down
```

次に、git を介して `superset` リポジトリをミラーリングするフォルダーを更新します:

```bash
git pull origin master
```

次に、コンテナを再起動すると、変更された Docker イメージが自動的にプルダウンされます:

```bash
docker compose up
```

## Superset を手動で更新する

ネイティブ インストールでスーパーセットをアップグレードするには、次のコマンドを実行します:

```bash
pip install apache_superset --upgrade
```

## メタデータ データベースのアップグレード

次のコマンドを実行してメタデータ データベースを移行します:

```bash
superset db upgrade
superset init
```

スーパーセットをアップグレードしてもチャートやダッシュボードは削除されませんが、アップグレードする前にベスト プラクティスに従い、メタデータ データベースをバックアップすることをお勧めします。本番環境をアップグレードする前に、ステージング環境でアップグレードし、ピーク時以外の使用時に最終的に本番環境をアップグレードすることをお勧めします。
