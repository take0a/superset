# PyPI から Superset をインストールする

<img src={useBaseUrl("/img/pypi.png" )} width="150" />
<br /><br />

このページでは、[PyPI で公開](https://pypi.org/project/apache_superset/) の `apache_superset` パッケージを使用して Superset をインストールする方法について説明します。

## OS の依存関係

Superset は、データベース接続情報をメタデータ データベースに保存します。そのために、接続パスワードを暗号化するために暗号化 Python ライブラリを使用します。残念ながら、このライブラリには OS レベルの依存関係があります。

**Debian と Ubuntu**

Ubuntu **20.04 および 22.04** では、次のコマンドを実行すると、必要な依存関係がインストールされていることが確認されます:

```bash
sudo apt-get install build-essential libssl-dev libffi-dev python3-dev python3-pip libsasl2-dev libldap2-dev default-libmysqlclient-dev
```

Ubuntu **20.04 より前** では、次のコマンドを実行すると、必要な依存関係がインストールされていることが保証されます:

```bash
sudo apt-get install build-essential libssl-dev libffi-dev python-dev python-pip libsasl2-dev libldap2-dev default-libmysqlclient-dev
```

**Fedora および RHEL 派生の Linux ディストリビューション**

`yum` パッケージ マネージャーを使用して次のパッケージをインストールします:

```bash
sudo yum install gcc gcc-c++ libffi-devel python-devel python-pip python-wheel openssl-devel cyrus-sasl-devel openldap-devel
```

CentOS および Fedora の最近のバージョンでは、`dnf` を使用して若干異なるパッケージ セットをインストールする必要がある場合があります:

```bash
sudo dnf install gcc gcc-c++ libffi-devel python3-devel python3-pip python3-wheel openssl-devel cyrus-sasl-devel openldap-devel
```

また、CentOS では、インストールを機能させるために pip をアップグレードする必要がある場合があります:

```bash
pip3 install --upgrade pip
```

**Mac OS X**

最新バージョンの OS X を使用していない場合は、アップグレードすることをお勧めします。これは、ユーザーが遭遇した多くの問題が古いバージョンの Mac OS X に関連していることが判明しているためです。更新後、最新バージョンの XCode コマンドライン ツールをインストールします:

```bash
xcode-select --install
```

システムにインストールされた Python の使用はお勧めしません。代わりに、まず [homebrew](https://brew.sh/) マネージャーをインストールしてから、次のコマンドを実行してください:

```bash
brew install readline pkg-config libffi openssl mysql postgresql@14
```

最新バージョンの Python をインストールする必要があります。Superset で公式にサポートされている Python バージョンのリストについては、[pyproject.toml](https://github.com/apache/superset/blob/master/pyproject.toml) ファイルを参照してください。[pyenv](https://github.com/pyenv/pyenv) (および [pyenv-virtualenv](https://github.com/pyenv/pyenv-virtualenv)) などの Python バージョン マネージャーを使用することをお勧めします。

また、`pip` と `setuptools` の最新バージョンがインストールされていることも確認しましょう:

```bash
pip install --upgrade setuptools pip
```

最後に、特定の Python パッケージを適切にビルドするには、LDFLAGS と CFLAGS を設定する必要がある場合があります。これらの変数は次のようにエクスポートできます:

```bash
export LDFLAGS="-L$(brew --prefix openssl)/lib"
export CFLAGS="-I$(brew --prefix openssl)/include"
```

これらは、pip インストール要件時に利用できるようになります。

## Python 仮想環境

Superset を仮想環境内にインストールすることを強くお勧めします。Python には、`virtualenv` が同梱されています。[pyenv](https://github.com/pyenv/pyenv) を使用している場合は、[pyenv-virtualenv](https://github.com/pyenv/pyenv-virtualenv) をインストールできます。または、`pip` を使用してインストールすることもできます:

```bash
pip install virtualenv
```

以下の方法を使用して仮想環境を作成し、アクティブ化できます:

```bash
# virtualenv is shipped in Python 3.6+ as venv instead of pyvenv.
# See https://docs.python.org/3.6/library/venv.html
python3 -m venv venv
. venv/bin/activate
```

または、pyenv-virtualenv を使用する場合:

```bash
# Here we name the virtual env 'superset'
pyenv virtualenv superset
pyenv activate superset
```

仮想環境をアクティブ化すると、インストールまたはアンインストールしたすべての Python パッケージはこの環境に限定されます。コマンド ラインで `deactivate` を実行すると、この環境を終了できます。

### Supersetのインストールと初期化

まず、`apache_superset` をインストールします。

```bash
pip install apache_superset
```

次に、必須構成 SECRET_KEY と FLASK_APP を定義します:
```bash
export SUPERSET_SECRET_KEY=YOUR-SECRET-KEY
export FLASK_APP=superset
```

次に、データベースを初期化する必要があります:

```bash
superset db upgrade
```

次のコマンドを実行してインストールを完了します:

```bash
# Create an admin user in your metadata database (use `admin` as username to be able to load the examples)
superset fab create-admin

# Load some data to play with
superset load_examples

# Create default roles and permissions
superset init

# To start a development web server on port 8088, use -p to bind to another port
superset run -p 8088 --with-threads --reload --debugger
```

すべてがうまくいけば、ブラウザで `hostname:port` (たとえば、デフォルトではローカルでは `localhost:8088`) に移動し、作成したユーザー名とパスワードを使用してログインできるはずです。
