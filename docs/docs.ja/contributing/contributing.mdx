# Superset への貢献

Superset は [Apache Software Foundation](https://www.apache.org/theapacheway/index.html) プロジェクトです。
Superset のコアコントリビューター（またはコミッター）は、主に以下のチャンネル（誰でも参加できます）でコミュニケーションを行っています:

- [Mailing list](https://lists.apache.org/list.html?dev@superset.apache.org)
- [Apache Superset Slack community](http://bit.ly/join-superset-slack)
- [GitHub issues](https://github.com/apache/superset/issues)
- [GitHub pull requests](https://github.com/apache/superset/pulls)
- [GitHub discussions](https://github.com/apache/superset/discussions)
- [Superset Community Calendar](https://superset.apache.org/community)

その他の参考資料:

- [Superset Wiki (code guidelines and additional resources)](https://github.com/apache/superset/wiki)

## オリエンテーション

Superset 関連パッケージを含むリポジトリの一覧は次のとおりです:

- [apache/superset](https://github.com/apache/superset) は、[pypi](https://pypi.org/project/apache_superset/) で配布されている `apache_superset` Python パッケージを含むメインリポジトリです。このリポジトリには、[superset-frontend](https://github.com/apache/superset/tree/master/superset-frontend) フォルダ以下に、Superset のメイン TypeScript/JavaScript バンドルと React アプリも含まれています。
- [github.com/apache-superset](https://github.com/apache-superset) は、Superset 関連の小規模ツール、フォーク、実験的なアイデアを管理するための GitHub 組織です。

## 貢献の種類

### バグ報告

バグを報告する最善の方法は、GitHub で問題を報告することです。以下の情報を含めてください。

- ご使用のオペレーティング システム名とバージョン。
- スーパーセットのバージョン。
- バグを再現するための詳細な手順。
- トラブルシューティングに役立つ可能性のある、ローカル環境に関する詳細情報。

Python のスタックトレースを投稿する際は、[Markdown ブロック](https://help.github.com/articles/creating-and-highlighting-code-blocks/) を使用して引用してください。

_GitHub Issues としてオープンされた機能リクエストは、ディスカッションに移動されますのでご注意ください。_

### アイデアや機能リクエストの提出

GitHub で [アイデアに関するディスカッションスレッド](https://github.com/apache/superset/discussions/categories/ideas) を開始するのが最適です。

- どのように機能するかを詳しく説明してください。
- 実装を容易にするため、スコープはできるだけ狭くしてください。
- これはボランティア主導のプロジェクトであり、皆様の貢献は誰にとっても歓迎されるものであることを忘れないでください :)

大規模な機能やコードベースの大幅な変更を提案し、それらの変更を推進するには、**Superset Improvement Proposal (SIP)** を作成してください。[SIP-0](https://github.com/apache/superset/issues/5602) のテンプレートを参照してください。

### バグを修正する

GitHub の問題を確認してください。`#bug` タグが付けられた問題は、実装を希望するすべての人に公開されています。

### 機能の実装

GitHub の Issue を確認してください。`#feature` タグが付けられた Issue は、実装を希望するすべての人に公開されています。

### ドキュメントの改善

Superset には、Superset 公式ドキュメントの一部として、docstring、`docs/*.rst`、あるいはブログ投稿や記事といったウェブ上の形式で、より充実したドキュメントが常に必要です。詳細については、[ドキュメント](/docs/contributing/howtos#contributing-to-documentation) をご覧ください。

### 翻訳の追加

英語以外の言語に堪能な方は、Superset の UI からテキスト文字列の翻訳にご協力いただけます。`superset/translations/<language_code>/LC_MESSAGES/messages.po` にある既存の言語辞書を参照することも、新しい言語の辞書を作成することもできます。
詳しくは [翻訳](howtos#contributing-translations) をご覧ください。

### 質問する

[StackOverflow](https://stackoverflow.com/)には専用の[`apache-superset`タグ](https://stackoverflow.com/questions/tagged/apache-superset)があります。質問の際は、このタグをご利用ください。

## コントリビューターの種類

Apache Software Foundation (ASF) のプロジェクトガバナンスモデルに従い、Apache Superset には特定のコントリビューターの役割が設けられています。

### PMCメンバー

プロジェクト管理委員会（PMC）メンバーは、PMCによって選出され、プロジェクトの管理を支援するメンバーです。PMCメンバーは、コミュニティ開発、リリース管理、プロジェクトガバナンスなど、プロジェクト全体の健全性維持に責任を負います。また、プロジェクトの技術的な方向性についても責任を負います。

ApacheプロジェクトのPMCの詳細については、https://www.apache.org/foundation/governance/pmcs.html をご覧ください。

### コミッター

コミッターとは、PMCによって選出され、コードリポジトリへの書き込み権限（コミット権限）を持つ人物です。コミッターはコード、ドキュメント、ウェブサイトを変更し、他者からの貢献を受け入れることができます。

コミッターとPMCメンバーの公式リストは[こちら](https://projects.apache.org/committee.html?superset)でご覧いただけます。

### 貢献者

貢献者とは、コード、テスト、ドキュメント、課題、議論など、あらゆる形でプロジェクトに貢献した人のことです。

> SupersetプロジェクトのPMCメンバー昇格に関するガイドラインは、こちらでご確認いただけます: https://github.com/apache/superset/wiki/Guidelines-for-promoting-Superset-Committers-to-the-Superset-PMC

### セキュリティチーム

セキュリティチームは、PMCメンバー、コミッター、非コミッターから選ばれたメンバーで構成され、セキュリティ問題への対応を担当します。

セキュリティチームの新メンバーは、PMCメンバーによる投票によって選出されます。チームへの参加を希望される場合は、private@superset.apache.org までメッセージを送信してください。ただし、チームは小規模で、セキュリティ問題の解決に重点を置く必要があります。そのため、リクエストは個別に評価され、チームの規模は比較的小規模に抑えられ、セキュリティに積極的に取り組む貢献者のみで構成されます。

このセキュリティチームは、[ASF脆弱性対応プロセス](https://apache.org/security/committers.html#asf-project-security-for-committers) に従う必要があります。

新しいセキュリティの問題はすべて、[ASF の JIRA スーパーセット セキュリティ プロジェクト](https://issues.apache.org/jira/secure/RapidBoard.jspa?rapidView=588&projectKey=SUPERSETSEC) で JIRA チケットとして追跡されます。

セキュリティチームのメンバーは、以下の義務を負います:

- Apache Software Foundation と締結した [ICLA](https://www.apache.org/licenses/contributor-agreements.html) を締結していること。
- セキュリティチームメンバーから明示的に許可された場合を除き（例：セキュリティチームが問題の診断と解決に外部の専門家の関与が必要であると同意した場合など）、保留中または未修正のセキュリティ問題に関する情報を、いかなる者（雇用主を含む）にも開示しないこと。

Apache Superset の特定バージョンのリリースを監督する貢献者であるリリースマネージャーは、デフォルトでセキュリティチームのメンバーとなります。ただし、セキュリティ問題の評価、議論、修正に積極的に関与することは期待されていません。

セキュリティチームのメンバーは、以下の一般的な期待事項にも従う必要があります。

- Superset におけるセキュリティ問題の評価、議論、修正、リリースに積極的に参加してください。
- 公開フォーラムでセキュリティ修正について議論することは避けてください。プルリクエスト (PR) の説明には、セキュリティ問題に関する情報を含めないでください。対応する JIRA チケットには、PR へのリンクを含める必要があります。
- 修正に貢献したセキュリティチームのメンバーは、CVE レポートに修正開発者として記載される場合があります。また、所属部署も記載する必要があります（所属部署の記載を希望する場合）。
